## 1. Purpose

Definir a arquitetura Retrieval-Augmented Generation (RAG) do NIS — como o conhecimento é recuperado do grafo e injetado no LLM para gerar respostas grounded.

## 2. RAG Pipeline

```
Clinical Query (observations + context)
    ↓
Query Embedding
    ↓
Vector Search (pgvector / dedicated vector DB)
    ↓
Top-K candidates (20)
    ↓
Re-ranking (clinical relevance + evidence grade)
    ↓
Filtered top-5 chunks
    ↓
Context assembly (structured + text)
    ↓
LLM inference (with guardrails)
    ↓
Structured output (JSON validated)
```

## 3. Knowledge Chunks

## 4. Retrieval Strategy

### 4.1 Hybrid Search

```
score = α × semantic_similarity + β × graph_proximity + γ × keyword_match

α = 0.50 (cosine similarity of embeddings)
β = 0.35 (graph distance from query nodes)
γ = 0.15 (exact keyword match)
```

### 4.2 Re-Ranking

Top-20 from vector search are re-ranked:

| Factor | Weight | Description |
|--------|--------|-------------|
| Clinical relevance | 0.40 | How relevant to the clinical scenario |
| Evidence grade | 0.25 | GRADE A > B > C > D |
| Population match | 0.20 | Matches patient population |
| Freshness | 0.15 | Newer evidence preferred |

### 4.3 Context Window Management

```
Total context budget: 8,000 tokens (model-dependent)
  - System prompt:     500 tokens
  - Clinical context:  1,000 tokens
  - RAG chunks:        5,000 tokens (top-5)
  - Guardrail prompt:  500 tokens
  - Output budget:     1,000 tokens
```

## 5. Guardrail Prompt

```
SYSTEM:
You are a nursing clinical reasoning agent. Follow these rules STRICTLY:

1. Use ONLY the NANDA, NIC, NOC codes provided in the context.
2. NEVER invent a code that is not in the context.
3. If no appropriate code exists, respond: "INSUFFICIENT_DATA"
4. Every diagnosis MUST include a probability from the provided Bayesian results.
5. Every recommendation MUST include evidence references from the context.
6. NEVER state certainty without probability.
7. Always include the disclaimer: "IA propõe, enfermeiro decide."
8. Output in structured JSON format only.

CONTEXT:
{rag_chunks}
{bayesian_results}
{patient_context}
{attention_filtered_observations}
```

## 6. Performance Targets

| Metric | Target |
|--------|--------|
| Retrieval latency | < 200ms |

| LLM inference latency | < 1.5s |
| Total RAG latency | < 2s |
| Retrieval precision | > 0.90 (top-5 relevant) |
| Hallucination rate | < 2% |
