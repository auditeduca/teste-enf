## 1. Purpose

Definir o uso de gRPC para comunicação interna de alta performance entre os serviços do NIS.

## 2. gRPC Services

| Service | Method | Purpose | Latency Target |
|---------|--------|---------|---------------|
| ReasoningService | RunInference | Execute reasoning pipeline | < 500ms |
| GraphService | Traverse | Graph traversal | < 50ms |
| RetrievalService | Search | Vector + graph retrieval | < 100ms |
| SafetyService | Check | Safety verification | < 30ms |
| MemoryService | Recall | Episodic memory lookup | < 100ms |

## 3. Protocol Buffers

```protobuf
service ReasoningService {
  rpc RunInference(ReasoningRequest) returns (ReasoningResponse);
}

message ReasoningRequest {
  string patient_id = 1;
  repeated Observation observations = 2;
  PatientContext context = 3;
}

message ReasoningResponse {
  repeated Hypothesis diagnoses = 1;
  repeated InterventionRecommendation interventions = 2;
  Explanation explanation = 3;
  float confidence = 4;
}
```

## 4. When to Use gRPC vs REST

| Scenario | Protocol | Reason |
|----------|----------|--------|
| Internal service-to-service | gRPC | Low latency, typed contracts |
| External API (EHR, partners) | REST/FHIR | Standard, widely supported |
| Real-time updates | gRPC streaming | Bidirectional streaming |
| Web frontend | GraphQL/REST | Browser compatibility |
