## 6. Sync Architecture

### 6.1 Inbound (External ÔåÆ NIS)

```
External EHR (FHIR)
    Ôåô FHIR subscription / polling
NIS FHIR Endpoint
    Ôåô Resource mapping
ni_interop.resource_mappings
    Ôåô Transform

    Ôåô Trigger

```

### 6.2 Outbound (NIS ÔåÆ External)

```
NIS Reasoning Engine produces recommendation
    Ôåô

    Ôåô

    Ôåô

    Ôåô
```

### 6.3 Bidirectional Sync

| Resource | Direction | Frequency | Conflict Resolution |
|----------|-----------|-----------|-------------------|
| Patient | Bi-dir | Real-time | NIS is source of truth for nursing |
| Encounter | Bi-dir | Real-time | EHR is source of truth |

| Condition (NANDA) | Outbound | On decision | NIS is source |
| Condition (CID) | Inbound | On update | EHR is source |
| CarePlan | Bi-dir | On change | NIS proposes, EHR confirms |

## 7. SMART on FHIR

### 7.1 OAuth Scopes

| Scope | Permission |
|-------|-----------|
| `patient/Observation.read` | Read patient observations |
| `patient/Condition.read` | Read patient conditions |
| `patient/CarePlan.read` | Read care plans |
| `user/Observation.write` | Write observations (nurse) |
| `user/Condition.write` | Write NANDA diagnoses |
| `user/CarePlan.write` | Write care plans |
| `user/ClinicalImpression.read` | Read reasoning traces |
| `launch/patient` | Patient context launch |

### 7.2 App Launch Flow

```
1. EHR launches SMART app with patient context
2. NIS receives launch token + patient ID
3. NIS requests OAuth token with scopes
4. NIS fetches patient data via FHIR
5. NIS runs reasoning pipeline
6. NIS writes back ClinicalImpression + CarePlan
7. NIS displays in SMART app within EHR iframe
```
