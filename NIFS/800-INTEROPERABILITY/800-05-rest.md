## 1. Purpose

Definir o padrão de API REST do NIS — os endpoints, payloads, versionamento e contratos que expõem o sistema ao mundo exterior.

## 2. API Design Principles

| Principle | Implementation |
|-----------|---------------|
| Resource-oriented | URLs nomeiam recursos, não ações |
| Versioned | `/api/v1/...` — breaking changes = v2 |
| Consistent | Mesmos padrões em todos endpoints |
| Documented | OpenAPI 3.1 spec gerada do NIFS |
| Rate-limited | 100 req/min default, configurável |
| Authenticated | JWT + SMART on FHIR scopes |

## 4. Payload Examples

### 4.1 Start Reasoning Session

```json
POST /api/v1/reasoning/sessions
{
  "patient_identifier": "hash_xxx",
  "case_id": "uuid",
  "reasoning_type": "diagnostic",
  "observations": [
    {"type": "assessment_score", "code": "BRADEN", "value": 12},
    {"type": "vital_sign", "code": "BP_S", "value": 90},
    {"type": "clinical_finding", "value": "total_immobility"},
    {"type": "context", "value": "ICU_postop"}
  ],
  "context": {
    "ward": "ICU",
    "population": "ICU",
    "shift": "morning"
  }
}
```

## 5. Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `page` | int | Pagination (default 1) |
| `limit` | int | Items per page (default 20, max 100) |
| `sort` | string | Sort field (-prefix for desc) |
| `filter` | string | Filter expression |
| `fields` | string | Field projection (comma-separated) |
| `include` | string | Include related resources |
| `locale` | string | Language (pt-BR, en-US) |

## 6. Error Responses

| Status | Error Code | Description |
|--------|-----------|-------------|
| 400 | `invalid_request` | Malformed payload |
| 401 | `unauthenticated` | Missing/invalid token |
| 403 | `forbidden` | Insufficient scope |
| 404 | `not_found` | Resource doesn't exist |
| 409 | `conflict` | Version conflict |
| 422 | `validation_error` | Semantic validation failed |
| 429 | `rate_limited` | Too many requests |
| 500 | `internal_error` | Server error (logged) |

```json
{
  "error": {
    "code": "validation_error",
    "message": "braden_score must be between 1 and 23",
    "field": "observations[0].value",
    "session_id": "uuid"
  }
}
```

## 7. Rate Limiting

| Tier | Rate | Burst |
|------|------|-------|
| Anonymous | 10/min | 20 |
| Authenticated | 100/min | 200 |
| Premium | 1000/min | 2000 |
| Internal | Unlimited | — |

Headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`
