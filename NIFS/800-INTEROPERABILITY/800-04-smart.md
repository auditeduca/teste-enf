## 1. Purpose

Definir como o NIS implementa SMART on FHIR para integração com EHRs hospitalares.

## 2. SMART App Launch Flow

```
1. EHR launches SMART app with patient context
2. NIS receives launch token + patient ID
3. NIS requests OAuth2 scopes:
   - patient/Observation.read
   - patient/Condition.read
   - patient/MedicationRequest.read
4. NIS fetches patient data via FHIR API
5. NIS processes data through reasoning engine
6. NIS writes back CarePlan resource via FHIR
```

## 3. SMART Scopes

| Scope | Access | NIS Use |
|-------|--------|---------|
| `patient/Observation.read` | Read vitals/labs | Feed reasoning engine |
| `patient/Condition.read` | Read diagnoses | Verify NANDA cross-map |
| `patient/MedicationRequest.read` | Read medications | Drug interaction check |
| `patient/CarePlan.write` | Write care plan | Push NIS recommendations |
| `launch/patient` | Patient context | Identify patient |
| `openid fhirUser` | User identity | Audit trail |
